Accessibiltiy evaluation reporting

As you are testing for accessibility barriers, assemble your findings into an accessibility evaluation report. You can use the accessibility evaluation template for starting a new report.

Writing about accessibility barriers

Within the accessibility evaluation report, write about each accessibility barrier, including where you found the barrier, what type of barrier it is, a description of the barrier, as well as who might be impacted by this issue.

Accessibility barrier reporting table

Page Barrier Description Users Impacted
Indicate which page(s) of the website or which part of the application the barrier occurs on. Give a brief summary of what type of barrier it is. Describe in greater detail what the barrier is and link to a screen shot (if applicable) of the barrier. Explain which users and in which scenarios users might be affected by this barrier.

Sample language for users impacted

  • Screen reader barriers: Users who are blind or have low vision and navigate via screen reader or other assistive technology
  • Color contrast barriers: Users who are colorblind or have low vision.
  • Keyboard navigation barriers: Users with motor disabilities who navigate via keyboard or other assistive technology
  • Magnification barriers: Users with low vision

High, medium, and low priority barriers

Accessibility evaluation reports break accessibility barriers into three categories: high, medium, and low priority.

  • High priority barriers prevent the user from accessing content, performing essential tasks, or navigating through the website, application, or program.
  • Medium priority barriers create significant obstacles to users being able to navigate, perform tasks, or access content.
  • Low priority barriers are an inconvenience for the user but are less urgent than high or medium priority barriers

Screenshots

When you encounter a barrier that has some visual element, take a screenshot to include in the report. For example, capture screenshots of magnification barriers, color contrast barriers, as well as screenshots that show any issues with the focus indicator while navigating via keyboard or with the screen reader on.

Linking to screenshots

Create a link from the barrier in your report to its respective screenshot by doing the following:

  1. Insert your screenshot into the screenshot section at the bottom of your document
  2. Create the appropriate level heading above the screenshot, such as "Figure 1: Home page heading has color contrast barriers."
  3. Return to the barriers table. In the description field for this barrier, write "See Figure 1"
  4. Link this text to the figure heading by creating a link, selecting the correct heading, and applying it to the text.

Writing alternative text

Add alt text when you include screenshots in your document. Learn more about how to write alt text from our accessible images and alt text guide.
