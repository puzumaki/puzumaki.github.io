Automated testing overview

Automated testing is when software scans code to detect accessibility issues. This is a good first pass for checking the web accessibility of a page.

Strengths

Automated tools can provide testing efficiencies and excel when the decision logic yes or no answers or uses machine-accessible data. They can help detect inaccessible page content by:

  • identifying issues that might otherwise be time consuming or difficult to identify manually;
  • catching missing alt text, form controls without labels, and contrast errors; and
  • flagging those machine-findable errors much better than a person can.

Limitations

Passing automated accessibility testing can create:

  • an assumption that they can replace human involvement,
  • a false sense of security if relied on a sole indicator of compliance, and
  • an inaccurate portrayal of site or app accessibility when people with disabilities, in fact, experience problems.
