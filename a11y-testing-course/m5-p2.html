Accessibility testing checklist

This is a complete list the auditing tests, condensed into a checklist for ease of access.

Automated scan

  • Run your preferred automated checker. Verify and document the issues that are flagged.

Keyboard navigation

  • Skip links: can you bypass repetitive navigation?
  • Focus order: can you navigate all interactive elements in logical order using Tab/Shift+Tab and have a visible focus state?
  • Keyboard access: can you Tab into and away from every component both forward and backward without getting stuck or causing unexpected behavior?

Visual design

  • Reflow: does content reflow at 320px width? Is content visible and usable in Chrome at 400% zoom without both horizontal and vertical scrolling at 1280px wide (or 200% text zoom in Firefox)?
  • Contrast: does normal text meet at least 4.5:1 contrast ratio including hidden content like modals, or does large text (18pt+ or 14pt bold), UI components, and graphics meet 3:1?
  • Color: is color used as the only visual means of conveying information?
  • Motion: can animations and auto-playing content be paused and it doesn't flash more than 3 times per second?

Content and context

  • Language: is the page language declared and language changes marked?
  • Images: is alt text meaningful and decorative images hidden?
  • Link purpose: do links describe their destination, with unique and descriptive labels for different locations?
  • Lists: are lists built with ordered and unordered list elements?
  • Video and audio: does media have captions or transcripts with 99% accuracy? Does video have audio that describes any visual content?

Screen reader tests

  • Page title: is it announced when the page loads?
  • Landmarks: are main, nav, header, footer regions identified?
  • Headings: can you navigate by heading? Is the hierarchy logical?
  • Tables: are headers associated with data cells?
  • Forms: are labels associated with inputs? Are errors announced? Are state changes (e.g. check a checkbox) communicated? Are required fields announced?
  • Error identification: are form errors clearly described?
  • Dynamic content: are updates announced via live regions? Does the focus move to revealed content, close when explicitly dismissed, and return the focus to the component that revealed it?

Optional additional tests

  • Voice control
    • Can you do all the things you can with a mouse using only your voice?
    • Can hidden content (e.g. hover tooltips) be accessed?
    • Do all the interactive elements have a number when you say “Show numbers”?
    • Do interactive elements respond to their labels?
  • Documents
    • Are PDFs accurately tagged in a logical order?
    • Do word documents pass accessibility checks?
  • Mobile application
    • Do two-fingered pinch zoom or three-fingered tap like gestures have alternate methods to operate?
    • Are any functions that can be operated by device motion such as shaking or walking also operable through UI components?
