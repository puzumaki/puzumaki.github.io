Keyboard navigation

Objective: Ensure the entire site can be navigated without a mouse.

Tools needed: Your computer keyboard. Tab to move forward and Shift + Tab to navigate backward.

The keyboard can only access interactive content.

Skip links

Skip links

Can you bypass repetitive navigation? The first link (or more) should be links to skip to other parts of the page, often “Skip to main content”. The link must be:

  • the first focusable element on the page,
  • visible on focus,
  • meet contrast requirements, and
  • go to the correct location on the page.

How to test

Keyboard

Keyboard

When I use the Tab key to move focus to a skip link, I see focus is strongly visually indicated.

Then when I use the Enter key to activate the link I, see my focus moves directly to the targeted element.

Desktop screen reader

Desktop screen reader

  • When I use a desktop screen reader (NVDA, JAWS, VoiceOver), and
  • I use the tab key to move focus to a skip link:
    • I hear it describes which landmark it's targeting.
    • I hear it identifies itself as a link.
    • I hear it is typically the first element in the page.
  • Then when I use the enter key to activate the link, I hear my focus moves directly to the targeted element.
Mobile screen reader

Mobile screen reader

  • When I use a mobile screen reader (Talkback, VoiceOver), and
  • I swipe to focus on a skip link:
    • I hear it describes which landmark it's targeting.
    • I hear it identifies itself as a link.
    • I hear it is typically the first element in the page.
  • Then when I double-tap with the link in focus, I hear my focus moves directly to the targeted element.

Focus order

Focus order

Interactive content must be:

  • focusable (can get to it with the keyboard): Assessment 2.1 Keyboard navigation,
  • visibly focused (the style must change with focus state): Assessment 3.1 Visible focus, and
  • able to move through the page in a logical order (often the reading order): Assessment: 3.5 Focus order.

How to test

Keyboard

Keyboard

  • When I use keyboard controls - Tab, arrow keys, Spacebar, Enter - to navigate through the page:
    • I see where the focus is on the page at all times (through a high contrast outline or underline), and
    • I see that the focus order follows the visual order or generally makes sense.

Keyboard access

Keyboard access

The keyboard should be able to move through and activate content that a mouse can.

  • The focus doesn’t get stuck (a keyboard trap): Assessment 2.2 No keyboard traps
  • The focus (tabbing) doesn’t cause any unexpected behavior (such as automatically submitting a form): Assessment 2.3 On focus

How to test

Keyboard

Keyboard

  • When I use keyboard controls - Tab, arrow keys, Spacebar, Enter - to navigate through the page:
    • I can perform all the actions I could with a mouse, and
    • I see that the keyboard does not get trapped in any area of the page.
