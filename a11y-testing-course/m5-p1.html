Evaluation reporting

As you are testing for accessibility barriers, assemble your findings into an accessibility evaluation report. You can use the "Issue list" tab on the accessibility evaluation template for starting a new report.

Accessibility barrier reporting table

Within the accessibility evaluation report, provide enough details so someone else can find and replicate the issue. The details to include correlate to columns D through J on the Issue List tab.

Give a brief summary of what type of barrier it is. Assign the priority as low, medium, or high. Describe in greater detail what the barrier is. Include details such as an HTML class or ID or color ratio. Optional: if possible, include a visual of the barrier. Optional: specify the WCAG criteria (it can help find the solution). The technology used is needed to replicate. If AT is used, include it as well. Indicate which page(s) of the website or which part of the application the barrier occurs on.

Issue Title

Use one to three words to summarize the issue. The title often aligns closely to the test name from the accessibility testing checklist (e.g. reflow) or the WCAG success criteria title, if known (e.g. info and relationships).

Priority levels

Accessibility evaluation reports break accessibility barriers into three categories: high, medium, and low priority.

  • High priority barriers prevent the user from accessing content, performing essential tasks, or navigating through the website, application, or program.
  • Medium priority barriers create significant obstacles to users being able to navigate, perform tasks, or access content.
  • Low priority barriers are an inconvenience for the user but are less urgent than high or medium priority barriers

Screenshots in description

When you encounter a barrier that has some visual element, take a screenshot to include in the report. For example, capture screenshots of magnification barriers, color contrast barriers, as well as screenshots that show any issues with the focus indicator while navigating via keyboard or with the screen reader on.

The best way to add a screenshot to the Google Spreadsheet is Insert > Images > Insert image in cell.

WCAG 2.1 criteria

Identifying criteria is optional but helpful. This column has an auto complete feature to help add success criteria. More detail about each success criteria is on the WCAG Criteria List tab.

Browser and assistive technology combo

Each browser, browser extension, and assistive technology interpret web content slightly differently. It's important to list the specific tech used to expose a barrier. For instance, perhaps you used Firefox to test for reflow, Safari with Voiceover to test headings, or Chrome with the Silktide extension to investigate landmarks.

Page URL

In most cases, copying the URL from the browser address bar is sufficient. However, if the URL obfuscates the location, adding a breadcrumb list for the page could also work (e.g. Home > About > Contact Us).
