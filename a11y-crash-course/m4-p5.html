Check PDF accessibility

Despite accessibility work done to a Google Doc or Microsoft Word file, you still need to perform an accessibility check and make any additional repairs to the PDF file as well.

Open the exported PDF file in Adobe Acrobat Pro (this is freely available to all students, faculty, and staff at the University of Minnesota). The software comes with a built-in accessibility checker.

Run the automated check

  1. From the Tools menu, select the Accessibility option then Accessibility Check.
  2. The Accessibility Checker Options window will open, select the Start Checking button at the bottom.
  3. The results panel will open with issues that need to be addressed. Right-click any issue and select Explain to get online documentation about the issue.

Help with errors

Document settings for title and language
  1. With the file open in Adobe Acrobat, go to File > Document settings
  2. In the Description tab, enter the document name in Title and other data if you would like (author, subject, keywords).
  3. Verify in the Initial View tab that the Window options "Show" is set to Document Title.
  4. Then in the Advanced tab, in Reading options make sure the Language dropdown is set to the correct language.
Accessibility tags

Some of the content in the PDF may not be tagged properly. These can be reviewed in the Tags panel, which is represented by an icon that looks like a price tag:

If you do not see the tag icon, add it to the interface by going to View > Hide/Show > Side Panels > Accessibility tags.

The content should be appropriately tagged. For instance, an <H1> is used for a Heading level 1; a paragraph is tagged with a <P>; etc. If there's something tagged that shouldn't be, either delete the tag or mark it as "artifact" by right-clicking on the item and selecting "Change tag to artifact...".

If you don't see anything in the tags panel, the document is not tagged. You can "autotag" the document using the 3-dot icon in the panel then "Autotag document". Having Adobe Acrobat tag the document means the results are likely to be less accurate and require more work.

If you have the original document, use a PDF export method that retains all the accessibility work that was done in that document.

Proper image tagging

All graphics need alternate text; this should have been included when creating the document, but if it wasn't retained when saved as a PDF it can be re-added.

In the Accessibility Tags panel, find the <Figure> tag.

Once you find the <Figure> tag, right-click to access the Properties. Under the Tags tab, add the alternate text in the "Alternate Text for Images" textbox.

Proper list tagging

Lists are fairly straightforward in a document editing app like Google Docs, they are a little messier in Adobe Acrobat. It has multi-layered tags for:

  • <L> is the List container
  • <LI> is an individual List Item
  • <Lbl> is the label, or the bullet character of the list item
  • <LBody> is the List Body, which contains the main content of the list item

Perform manual checks

Two document issues require a manual check:

  • Logical reading order: Adobe can't tell if the reading order is logical, that requires human reasoning.
  • Contrast: make sure the document's content follows contrast guidelines. If this was checked in the word document, the PDF should be okay.
Check reading order

You can check the reading order that assistive technology will navigate by using the Order feature. The icon looks like a Z:

If you do not see the icon, add it to the interface by going to View > Show/Hide > Side Panels > Order. Order should have a check next to it.

The Order icon will open a reading order panel and overlay numbers on each object in the document. Check to make sure the items you expect are included and are read in the order you expect.
