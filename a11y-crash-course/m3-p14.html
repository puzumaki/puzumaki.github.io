Skill 5: Lists

How-to resources

  • UMN Lists
  • Write parallel list items that have the same structure.

Lists help users comprehend text more quickly, can provide a break in the document flow, and encourage users to stick with the content. They help reduce reader fatigue resulting from trying to comprehend dense or complex paragraphs.

Use the linked resource on this page, listed under the “How-to resources” header, for relevant tutorials and how-to guides.

Lists checklist

  • Use lists to organize information into meaningful sections.
  • Use lists to convey sequencing or steps.
  • Ensure lists are grammatically parallel—start each item with the same part of speech (adjective, verb, or noun).
  • Format lists using styling or proper coding; don’t manually insert numbers, characters, or images to create a list.
  • Use standard bullet styles in Microsoft Word and Google Workspace products.
  • Structure long ordered lists using a different list style in the secondary levels. This is often automatic in many document tools.
  • Section lists logically and condense information in ways that are meaningful.

Details

Use lists to facilitate scanning
  • Use lists to present key terms and concepts, organize information into meaningful chunks, convey sequencing necessary for processes and procedures, or create links to additional information.
  • Don't use list formatting just to draw attention to an item. Lists should always contain list items.
  • Don't format unlike items as a list unless you rewrite them so they have the same structure.

Examples

Do: Use lists for multiple items that make sense together:

  • Item one
  • Item two
  • Item three

Don't: Use list formatting on a single item to draw attention to it:

  • Very Important Item Number One (there are no items beyond item number one).
Format lists correctly

If you don't format your lists correctly, adaptive technology users won't be able to identify the text as a list.

  • Use the list or paragraph formatting options in your document or presentation software or HTML list tags on a webpage.
  • Use a bulleted list to show a list of related but non-sequential items. This type of list is called an unordered list.
  • Use a numbered list to show steps in a process or the number of parts in a whole.
  • Don't create lists manually by simply inserting numbers, characters, images, or other symbols before list items and then indenting the item text.
  • Don't add extra carriage returns after list items because that will add unnecessary code to your document or webpage.

Examples

Do: use an unordered list of items related to one another but not sequential:

  • Item one
  • Item two
  • Item three

Don't: use an ordered lists for items related to one another but not sequential:

  1. Item one
  2. Item two
  3. Item three

Do: use numerical lists for a sequential process:

  1. Step one
  2. Step two
  3. Step three

Don't: use bulleted lists for a sequential process:

  • Step one
  • Step two
  • Step three
Keep lists simple

Although screen readers can detect bulleted (unordered) lists, they do not usually distinguish levels of indentation. Here are some tips for how to handle long lists:

  • Avoid listing items in a long list if possible.
  • Try to structure your long lists into smaller lists separated by headings.
  • Avoid nesting lists (creating lists with multiple levels of indentation).
  • If you must create a nested ordered list, use different numbering systems in the primary and secondary levels.
  • If you must create a nested unordered list, use different bullet types in the primary and secondary levels.
Use standard bullets

When working with standard office software, such as Microsoft Office or Google Suite, always use the built-in generic bullet characters. Replacing the generic bullet characters provided by Microsoft Office and Google Suite products (e.g., circles and squares) with custom characters could result in an image being included without alt text or in a screen reader interpreting a character by its ASCII value.

On websites, you can replace standard bullets with custom images using CSS. However, the bullet will be read by a screen reader as if it were unstyled.

Sentence structure for lists

If each bullet stands alone, use proper sentence structure for each bullet.

If the bulleted list is an extension of a sentence, ensure that each bullet is:

  • not started with a capitalized letter,
  • readable as if the sentence only contained that one bullet,
  • joined together with punctuation as if a compound group, and
  • ended by tying the last bullet with a conjunction, such as "and".

Optional: More tips for readability

Lists can greatly improve content readability, typography decisions also have impact. Typography is the art of making text readable and appealing to the eye. Here are a few guidelines that can affect the accessibility of your content:

Choose sans-serif fonts

Sans-serif fonts may be easier to visually process on screens, and more common fonts may help people with dyslexia, low vision, or visual processing difficulties (because of the amount of exposure and experience with them). Using a serif font is not considered an accessibility issue, and automatic accessibility checkers won't check whether your font is serif or sans-serif. However, choosing a sans-serif font can make your document easier to use.

Read WebAIM's how fonts affect accessibility. Note that research is ongoing on for typography and readability.

Use left-aligned text

Justified text is more difficult to process because of extra space between the words.

Centered text is challenging for readers with dyslexia because of the jagged way in which it moves the eye around the text.

Use larger font sizes

Use font sizes of at least 12pt for print documents and digital documents people will collaborate in, and at least 14pt if you plan to screenshare your document, such as via Zoom.

Use italics only for intended purposes

Italics make letters seem like they are encroaching on adjacent spaces, creating a barrier to some readers. Use italics sparingly and only for short phrases at most. They are used for emphasis, if it's needed to add clarity to the sentence, or to distinguish phrases such as the Latin name of a species in a sentence of text. They should never be used for full sentences or captions.

Do not rely on visual formatting alone to communicate meaning

Screen readers might not announce formatting changes, such as bold or highlighting; readers with color-blindness may not perceive changes in text color. To mark an important section of text, add a signal word like Important or Note.
