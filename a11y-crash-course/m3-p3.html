Alternative text, or “alt” text, describes the visual content of images, graphs, and charts. Alternative text can be read by assistive technologies; it also can help you make sure an image communicates your intended purpose.

There are two groups of people you need to consider when using or creating images in digital content:

  • Those who can see the images.
  • Those who can't.

It may seem obvious that you'll need to provide the information in an alternative format for people who can't see your visuals—those who are blind or visually impaired—but sighted users who disable image loading or encounter a broken image also won't know what the image is meant to convey.

Use the linked resources on this page, listed under the “How-to resources,” for relevant tutorials and how-to guides.

Key point: Alternative text is required for all graphics, even for graphics that have no purpose otherwise known as decorative. Decorative graphics may have empty alt text, but the "alt" is present either by marking it as decorative in an application or leaving the "alt" HTML attribute in code.

Alternative text checklist

  • Only include images that add to the meaning of the content.
  • Write concise alt text that is one to two sentences long and ends with a period. (Best practice suggests no more than 125 characters if possible.)
  • Consider adding a longer explanation below the image in addition to alt text that summarizes the meaning if your image is complex and needs further explanation.
  • Don't say "picture of" or "photo of", but do describe whether it's a piece of art, logo, illustration, or cartoon.
  • Alt text is context dependent, so write unique alt text for the same image if used in different contexts.

Details

Keep it brief and contextual

Write in simple, precise language and keep the explanation brief and contextual. Typically no more than a few words are necessary, though on rare occasions, a short sentence or two may be appropriate.

  • Don't simply describe the surface features of the image or graphic.
  • Don't include the same alt text everywhere an image is used.
  • Don't use the name of the file itself (e.g., “graphic.png”).
  • Avoid the use of phrases such as “image of ...” or “graphic of ...”; most users will be aware that an image is present, either by visual observation or adaptive technology.
  • If the information about the graphic is located elsewhere in the document, such as part of the body text or in an image caption, you don't need to write full alt text.

Note: Longer descriptions (more than about 125 characters) should be included in the body text of your document or as a separate page that you link to, rather than as alt text.

Mark decorative or contextualized images properly

Decorative objects add visual interest but aren't informative (for example, stylistic borders). When you mark decorative objects correctly, people using screen readers will hear that these are decorative, so they know they aren't missing any important information.

You do not need to write alt text for images and graphics that are purely decorative, such as a graphic of a line break, or when the same information you would put in the alt text is provided in the surrounding content.

However, you should mark the image as decorative or indicate that the alt text for that image should be blank. For example, some content apps like Microsoft Word or PowerPoint allow you to check a box to indicate that an alt description is not necessary, or an image is decorative. Others may require that you enter a null attribute instead (usually a "").

Technical note: There is a difference between an empty alt (alt="") and no alt. The empty alt lets assistive technologies know that an image is there, and allows people to skip it. If there is no alt, then the webpage or document may behave in unexpected ways like read out the image URL.

  • Don't include alt text with images that are purely decorative (leave the alt empty).
  • Don't include alt text that provides the same information as text within the context of the image, such as in the caption (this is redundant).

Note: Challenge yourself to explain the content and context of an image, even if you initially think it is decorative. If it truly carries no meaning, consider removing it entirely.

Provide a description for charts and graphs

You should add alt text for all charts and graphics derived from data tables, such as from spreadsheets. However, charts and graphs made from data tables in spreadsheets usually require more complex descriptions than alt text can provide. There are several ways to provide longer descriptions so that all users will benefit.

  • Don't provide descriptions for charts and graphs in a spreadsheet if simple alt text will suffice.
  • Don't provide a description for charts and graphs in a spreadsheet if users have access to the source file from which the chart or graph was created.
  • If a longer description is necessary, insert the description as visible text next to the graph or chart.
  • If your description is too long to fit in alt text or next to your chart or graph, include a link to a separate page with your longer description.
  • When making a chart or graph in Microsoft Excel or Google Sheets, be sure to fill in fields such as title, labels, values, relationships among variables, x and y axes, and scale increments,
  • Test your alt text and descriptions by having someone review the content with the images replaced by the alt text or descriptions.
Label icons

An icon is a small picture or symbol, often used to represent a logo or object. If you use icons in your document:

  • Label each icon. Even if an icon's meaning seems unmistakable to you, people with different cultural backgrounds and processing styles frequently interpret icons very differently from their designers.
  • Use icons consistently. If you use an icon in one area of your document, repeat it throughout your document.
